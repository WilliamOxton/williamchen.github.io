问题 1：GitHub Pages 域名
结论：命名对了基本自动生效，但有几个细节要注意。
命名规则
你的 GitHub 用户名决定域名：
代码用户名: abc仓库名: abc.github.io   ✅ 正确访问域: https://abc.github.io

⚠️ 仓库名必须严格等于 你的用户名.github.io，否则只能访问 abc.github.io/仓库名/ 这种子路径。

需要做的设置（不多）


创建 repo，命名 abc.github.io，设为 Public


进入 Settings → Pages


Source 选择 Deploy from a branch，Branch 选 main，路径选 /root


保存后等 1-2 分钟，自动部署


不需要额外注册任何东西，GitHub 自动识别这个命名约定。

问题 2：前端自动导出 PDF 简历
完全可以做，而且有几种方案可选：
方案 A：Print CSS + window.print()（推荐，最简单）
原理：写一套专门针对打印的 CSS，用户点击按钮触发浏览器打印，选"另存为PDF"。
CSS/* 正常页面样式 */@media screen {  .navbar, .hero-section { display: block; }}/* 打印/PDF 专用样式 */@media print {  .navbar, .sidebar, .download-btn { display: none; }  /* 隐藏不需要的元素 */  .resume-section { display: block; font-size: 11pt; }  @page { margin: 1.5cm; size: A4; }}
JavaScriptdocument.getElementById('download-pdf').addEventListener('click', () => {  window.print();});
优点：零依赖、字体清晰、文字可复制
缺点：最终效果依赖用户浏览器，不能100%控制输出

方案 B：html2pdf.js（控制更精确）
HTML<script src="https://cdnjs.cloudflare.com/ajax/libs/html2pdf.js/0.10.1/html2pdf.bundle.min.js"></script>
JavaScriptdocument.getElementById('download-pdf').addEventListener('click', () => {  const element = document.getElementById('resume-content');    html2pdf().set({    margin: [10, 10],    filename: 'YourName_CV.pdf',    image: { type: 'jpeg', quality: 0.98 },    html2canvas: { scale: 2, useCORS: true },    jsPDF: { unit: 'mm', format: 'a4', orientation: 'portrait' }  }).from(element).save();});
优点：输出完全可控、文件名自定义
缺点：本质是截图转PDF，文字有时不可复制（学术场景略劣）

方案 C：预生成 PDF 直接提供下载（最稳定）
直接把 PDF 文件放进 repo：
HTML<a href="/assets/YourName_CV.pdf" download="YourName_CV.pdf">  <button>Download CV (PDF)</button></a>
优点：100%保真，无任何技术风险
缺点：简历更新后需手动重新上传 PDF

我的建议
场景推荐方案想省事、质量高方案 C（预放PDF）+ 方案 A 作为备用想全自动、内容动态方案 A（Print CSS）追求精确排版控制方案 B（html2pdf.js）
最实用的做法：同时放一个静态 PDF 下载链接（方案C）+ 做好 Print CSS（方案A），两者互补，覆盖所有场景。
